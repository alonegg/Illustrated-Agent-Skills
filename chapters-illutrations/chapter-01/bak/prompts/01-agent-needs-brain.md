插图位置：第 1 章 - 1.1 准备工作环境 - 图示 1
本段内容摘要：智能体需要连接大模型才能工作，就像汽车需要发动机一样。不同路径获取“大脑”的方式不同——扣子自带模型，Claude Code 需要订阅，OpenClaw 需要 API Key。
读者要获得的“瞬间理解”：智能体 = 车身，大模型 = 发动机，缺一不可。
推荐布局：hub-spoke
配图作用：概念具象化

画面整体：中心是一个大的圆角矩形，内含一个简化的机器人/智能体图标（用齿轮 + 对话气泡组合表示），代表“智能体”。这个中心节点用 #D87757 高饱和色填充，是画面绝对焦点。

从中心向外辐射出三条黑色细连接线，连接到三个均匀分布的小圆角矩形子节点：

【子节点一（左上）】
一个简化的豆荚/扣子图标，旁边标签写“扣子”。节点内有一个小的大脑图标已经嵌入其中（暗示自带模型），用 #D87757 中透明度填充。

【子节点二（右上）】
一个简化的终端窗口图标，旁边标签写“Claude Code”。节点内有一个钥匙图标（暗示需要订阅/API Key），用 #D87757 中透明度填充。

【子节点三（下方）】
一个简化的龙虾图标（OpenClaw 的象征），旁边标签写“OpenClaw”。节点内同样有一个钥匙图标，用 #D87757 低透明度填充。

中心节点上方浮着一个大脑图标（代表大模型），用黑色轮廓线绘制，与中心节点之间有一条粗虚线连接，暗示“智能体需要连接大脑”。

次要元素：

1. 中心节点外有 #D87757 低透明度的光环衬托重要性
2. 三条连接线上各有一个小箭头，从中心指向子节点
3. 画面右下角空白处可有一组极简的“车身 + 发动机”小图示作为类比暗示
