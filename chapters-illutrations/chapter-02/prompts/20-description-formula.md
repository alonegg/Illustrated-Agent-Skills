一、插图输入
插图位置：第 2 章 - 2.4.3 如何写出好的 description - 极简公式
本段内容摘要：描写好 description 的极简公式是：description = 功能定义 + 触发场景/触发词。功能定义——这个技能是用来干什么的；触发场景/触发词——什么时候使用这个技能。description 需要贴合用户实际表述，尽可能覆盖用户可能使用的关键词。
读者要获得的“瞬间理解”：description 由两个核心组成部分构成——“功能定义”（做什么）+ “触发场景/触发词”（什么时候用）。
推荐布局：structural-breakdown
配图作用：概念具象化
主要元素：画面居中一个大的手绘圆角矩形盒子，标注“description”，盒子用大括号或分隔线拆解为两个并列的小盒子：左侧小盒子“功能定义”、右侧小盒子“触发场景/触发词”
次要元素：两个小盒子之间有一个手绘“+”号连接；整个 description 大盒子外用“=”“描述技能”的等号示意公式关系；左侧小盒子下方标注小字“做什么”，右侧小盒子下方标注小字“什么时候用”
需要上屏文字：

* 大盒子标签：“description”
* 两个小盒子标签：“功能定义”、“触发场景/触发词”
* 下方小字注释：“做什么”、“什么时候用”
* 整体公式：description = 功能定义 + 触发场景/触发词（可用手写感方程式排列）
  禁用元素：真实代码截图、写实人物、复杂纹理、密集文字、章节标题

二、图片比例
16:9 横向。

三、全书统一风格
手绘涂鸦风插画——像用马克笔在白板或牛皮纸上随手画的。线条不求笔直，带点手抖感、断续感；字迹带手写感但清晰可读；框线随意弯曲，形状略带不规则；连接线/箭头自然弯曲，不用直角。整体轻松、有温度、不正式，像“朋友在白板上帮你画的图”。干净构图，大量留白；2D；无阴影或极少阴影。

四、配色方案（出版三色限定，不可更改）
主色：#D87757（暖赤陶色）—— 主要视觉元素、强调色块。
黑色 —— 轮廓线、文字、屏幕脸、深色区域。
白色 —— 背景、留白、面板、关节点缀。
仅使用以上三色及其深浅变体（通过透明度/灰度过渡），不使用其他色相。

五、布局视觉语法（structural-breakdown 结构拆解图）

* 主体对象（description 大盒子）居中偏上，占画面主要空间
* 从主体的两个部分（功能定义 / 触发场景）分别引出标注线，指向下方小盒子或周围的说明标签
* 标注线用细实线 + 小圆点起始
* 两个小盒子用不同透明度的 #D87757 底色区分
* 整体像公式的分解示意图，保持手绘风格

视觉元素

* 主体对象：一个大的 #D87757 圆角矩形，清晰标注“description”，内部用细虚线分为左右两半
* 两个小盒子：分别用 #D87757 深浅不同透明度区分，功能定义（深）+ 触发场景（浅），或两者均匀配色
* 连接标注线：黑色细线，从主盒子的两个部分引到下方小盒子
* 公式符号：醒目的“=”（在 description 左侧）、“+”（在两个小盒子之间）手绘感符号

五点五、信息层次
主要元素（最大最醒目）：中心 description 大盒子和其拆解出的两个小盒子
次要元素：两个小盒子下方的小字注释（“做什么”、“什么时候用”）
视觉权重：description 盒子为视觉焦点，两个组成部分左右均衡。

六、构图要求
公式感排布：description = 功能定义 + 触发场景/触发词；整体像一个手绘黑板上的简洁方程式；留白用于呼吸；避免拥挤。

七、文字处理
大盒子中央：`description`（可用代码字体或手写体）
左侧小盒子：`功能定义`，下方小字“做什么”
右侧小盒子：`触发场景/触发词`，下方小字“什么时候用”
公式符号：“=”、“+”
中文标签字体使用 Noto Sans CJK SC。description 一词可保留英文。其余文字使用中文。严禁在图片中出现章节标题或“第 X 章”等字样。

八、技术与限制
所有图像与文字都保持手绘/涂鸦感，避免写实元素、照片风、死板直角连线和企业风流程图；不出现品牌 logo；不使用真实界面截图风；线条可以不完美、有断续感；画面简洁突出关键词与核心概念。

九、最终生成目标
让读者在不读正文的情况下，3 秒看懂：description 的极简公式——由“功能定义”（做什么）+ “触发场景/触发词”（什么时候用）两部分组成。
