插图位置：第 2 章 - 2.4.1 智能体如何“相中”技能的 - 图示 6
本段内容摘要：用户发出指令后，智能体拿指令和所有技能的 description 做比对，找到最匹配的技能，然后才读取该技能的 SKILL.md 正文。description 是大模型的唯一决策依据，它的最重要读者是 AI，不是人。
读者要获得的“瞬间理解”：用户说一句话 → 过滤所有技能名片 → 只有匹配的那个被“选中”加载。
推荐布局：funnel
配图作用：概念具象化

画面整体：从上到下三级漏斗结构，体现从“多”到“一”的筛选过程。

【入口区域（漏斗上方）】
画面顶部有一个对话气泡图标，里面写“帮我写周报”（代表用户指令）。气泡下方散落着 5-6 个大小一致的小卡片图标（代表所有已安装技能的 description 名片），用黑色低透明度绘制。这些小卡片上各有 1-2 行极简横线，暗示“不同技能的描述”。

【第一级漏斗 —— 语义比对】
一个较宽的圆角横条，用 #D87757 低透明度填充。横条上有一个放大镜图标（暗示“大模型在做语义比对”）。旁边标签写“语义比对”。从上方散落的卡片经过这一层后，大部分被筛掉（灰色虚线表示“不匹配”），只有 2-3 张通过。

【第二级漏斗 —— 相关性排序】
一个中等宽度的横条，用 #D87757 中透明度填充。横条上有一个排序/星星图标。旁边标签写“相关性排序”。通过上一层的 2-3 张卡片在这里被排序，最相关的排在最前。

【出口区域（漏斗下方）—— 命中】
最窄的出口，用 #D87757 高饱和色高亮。只有一张卡片从出口出来，这张卡片上标注“周报技能”，卡片下方连接着一个翻开的文档图标（代表 SKILL.md 被读取）。旁边标签写“加载 SKILL.md”。

次要元素：

1. 被筛掉的卡片用灰色虚线标记，淡出到两侧
2. 漏斗出口处有一个小箭头指向文档图标，暗示“匹配后才加载正文”
3. 漏斗右侧可有一个简化的大脑图标，连线指向漏斗，暗示“这个过程由大模型完成”
