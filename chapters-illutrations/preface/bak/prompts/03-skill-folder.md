插图位置：第 0 章 - 技能文件夹结构
本段内容摘要：技能的本质是把个人经验浓缩为可执行、可复用的操作手册。形式上就是一个普通文件夹，核心是一份 SKILL.md 说明书，再配上可选的脚本和参考文档。结构极其简单。
读者要获得的“瞬间理解”：技能不是什么高深的东西，就是一个文件夹加一份说明书。
推荐布局：structural-breakdown
配图作用：信息补充

画面整体：用可视化的目录树结构展示技能文件夹的组成，像开发者在终端中看到的文件树，但以插画风格呈现。

【主体：目录树结构】
画面中央偏左展示一个清晰的文件目录树，用树形缩进线连接：

```
my-skill/
├── SKILL.md        ← 核心说明书
├── scripts/        ← 可选脚本
│   └── main.ts
└── references/     ← 可选参考文档
    └── style-guide.md
```

视觉处理：

* 根目录 `my-skill/` 用一个 #D87757 高饱和的文件夹图标表示，最大最醒目
* `SKILL.md` 是整棵树中最突出的节点：用 #D87757 高饱和色的文件图标，旁边带一个星号或闪光标记，表示“这是核心”
* `scripts/` 和 `references/` 用 #D87757 低透明度的文件夹小图标，体积明显小于根目录
* 树形连接线用黑色细线，清晰展示层级关系
* 每个节点旁有对应的文件/文件夹名称文字

【右侧标注】
从目录树的三个关键节点引出标注线，指向右侧的标签卡片：

* SKILL.md → 标签“说明书”，下方小字“核心”
* scripts/ → 标签“脚本”，下方小字“可选”
* references/ → 标签“参考文档”，下方小字“可选”

标签卡片用小圆角矩形，SKILL.md 的标签用 #D87757 高饱和底色（白色文字），其余两个用 #D87757 低透明度底色。

次要元素：

1. 标注线用黑色细实线 + 小圆点起始
2. 背景大面积留白，突出目录结构的简洁感
3. 整体传达“就这么简单”的轻松感
