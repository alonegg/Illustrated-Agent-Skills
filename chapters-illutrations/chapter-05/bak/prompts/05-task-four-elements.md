插图位置：第 5 章 - 5.3.2 - 图示 5
本段内容摘要：给子智能体的任务描述需要四个要素：目标（交付物是什么）、约束（必须遵守的风格/禁止清单）、输入（要读取哪些文件路径）、验收（交付前自检条件）。四个要素全齐了，子智能体不需要追问就能开工。
读者要获得的“瞬间理解”：派活写清四要素 = 目标 + 约束 + 输入 + 验收
推荐布局：structural-breakdown
配图作用：信息补充

画面整体：一张大的“派工单”卡片居中偏左，从卡片四个区域引出标注线指向右侧标签。

【主体 - 派工单卡片】
一个大的圆角矩形，用黑色细边框描边，内部分为四个水平区域，用极细虚线分隔。卡片顶部有一个标题区写“任务描述”，用 #D87757 底色。

【区域一（上部）- 目标】
\#D87757 低透明度底色。内含一个靶心图标（极简线条）和短文字“draft-a.md”。从此区域引出标注线，指向右侧标签卡片“目标”，标签下方极小字“交付物 + 格式”。

【区域二 - 约束】
白色底色。内含一个围栏/护栏图标和短文字“风格 · 禁止”。从此区域引出标注线，指向右侧标签卡片“约束”，标签下方极小字“规则 + 禁止清单”。

【区域三 - 输入】
\#D87757 极低透明度底色。内含一个文件夹图标和几个小文件名“source.md”“analysis.md”。从此区域引出标注线，指向右侧标签卡片“输入”，标签下方极小字“文件路径”。

【区域四（底部）- 验收】
白色底色。内含一个勾选清单图标和三个小勾。从此区域引出标注线，指向右侧标签卡片“验收”，标签下方极小字“自检 3 条”。

次要元素：

1. 标注线用黑色细线 + 小圆点起始
2. 右侧四个标签卡片用 #D87757 不同透明度的小圆角矩形
3. 整体布局紧凑但不拥挤，留白充足
4. 卡片左上角可有一个小图钉或回形针装饰
