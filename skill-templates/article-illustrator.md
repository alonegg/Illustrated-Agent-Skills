# 文章配图技能

## 工作流程

### 步骤一：分析配图需求

逐段分析文章，识别需要配图的位置。

**适合配图的内容**：
- 抽象概念需要可视化
- 流程/步骤需要图示
- 对比关系需要可视化
- 核心观点需要强化

**不需要配图的内容**：
- 代码示例
- 简单列表
- 已经很直观的描述

**数量原则**：宁可少而关键，也不要多而分散。默认每个主要章节 1 张，只在确实需要辅助理解时再加。

### 步骤二：选择风格

根据文章主题选择统一风格：

| 风格 | 适用场景 |
|------|---------|
| 科技感 | AI、算法、数据相关话题 |
| 手绘涂鸦 | 轻松、思考类内容 |
| 极简扁平 | 流程、对比、通用话题 |
| 信息图表 | 数据、流程、对比类内容 |

同一篇文章内保持风格一致。

### 步骤三：生成配图计划

为每个配图位置列出：
- 插入位置（哪个段落之后）
- 配图目的（为什么这里需要图）
- 视觉内容（图片应该展示什么）
- 文件名（小写英文 + 连字符）

### 步骤四：生成图片

为每张图：
1. 撰写详细提示词，保存到 `imgs/prompts/` 目录
2. 调用图像生成工具
3. 生成失败自动重试一次
4. 保存到 `imgs/` 目录

### 步骤五：插入文章

将图片插入对应位置：

- 格式：`![简洁描述](imgs/文件名.png)`
- 图片前后各留一个空行
- alt 文本用中文简洁描述

## 文件结构

```
article-folder/
├── article.md
└── imgs/
    ├── prompts/
    │   ├── concept-comparison.md
    │   └── workflow-diagram.md
    ├── concept-comparison.png
    └── workflow-diagram.png
```

## 注意事项

- 配图服务于内容，避免为了配图而配图
- 文件名用有意义的英文描述，不用序号
- 敏感人物使用卡通替代形象
- 同一篇文章内保持风格一致
